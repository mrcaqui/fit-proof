ゲーミフィケーションについても、`submission_items`や`submission_rules`のように削除済項目を設けて`effective_to`の概念で設定変更が過去に影響しないようにしたいです。

ゲーミフィケーション設定は`profiles`の`gamification_settings`にjsonbとして保存されているので、この変更に伴いこのままで良いのか、新たなテーブルを設けた方が良いのか、シンプルさと運用の観点で推奨を提案して下さい。

表示/非表示(`enabled`)は特に過去への影響は無くUI上の表示だけの問題なので変更の対象に含めなくてOKです。
`effective_from`はゲーミフィケーションのための計算を開始する起点を示しているだけなので、この変更により不整合はでないと思いますが、懸念点あれば教えて下さい。

対象は以下になります。

{
"shield": {
"condition_type"
"straight_count"
},
"straight": {
"allow_shield"
"allow_revival"
"use_target_days"
"custom_required_days"
},
}

また、このタイミングで容易に追加できそうであれば追加してほしいですが、`ストレート達成時に共用する項目`について、「期限超過を達成としてカウント」を新設し、有効の場合には該当週に`is_late`が`TRUE`のSumissionが1つでもあればストレートとしてカウントしない、というトグルを追加してほしいです。
